# Contributing to git-architecture-preso

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make your changes
5. Commit with descriptive messages
6. Push and create a pull request

## Commit Message Format

All commits must include project tracking tags:

```
Brief summary [project:card=Issue Title]

- Bulleted reason 1
- Bulleted reason 2
- Bulleted reason 3

[project:status=In Progress|Done]
```

**Status values:** `Todo`, `In Progress`, `Done`

## Code Style

- Follow language-specific best practices
- Write clear, descriptive commit messages
- Add tests for new features
- Update documentation as needed

## Issues

Before opening a new issue:
- Search existing issues to avoid duplicates
- Use the appropriate issue template
- Provide clear reproduction steps for bugs
- Describe expected vs actual behavior

## Pull Requests

- Keep PRs focused on a single issue/feature
- Update tests and documentation
- Ensure CI checks pass
- Reference related issues

---

For questions, open an issue with the `question` label or reach out to Umut Topkara.
